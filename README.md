# Gemma4 LoRA 学習：孫悟空口調ファインチューニング

Gemma4 を LoRA（Low-Rank Adaptation）で学習させ、孫悟空のような口調で応答するモデルを作成しました。

## 概要

- **ベースモデル**: Gemma4
- **学習手法**: LoRA
- **目的**: ユーザーの質問に対して、孫悟空風の口調で回答するモデルを構築

## リポジトリ構成

```
Gemma_lora_project
├── training_data_goku_1k.csv
├── train.py
├── requirements.txt
└── README.md
```

## データセット
学習データは [`data/train_data.csv`](./data/train_data.csv) を使用しています。

- ユーザーの質問と、それに対する孫悟空口調の回答をペアにしたデータセットを **1,000 件** 準備
  - 学習用: 900 件
  - 評価用: 100 件

### データ形式
| question | answer |
|---|---|
| ズルして楽する方法は？ | ズルはダメだ、オラは大っ嫌いだ！楽して得た力なんて、いざってときに何の役にも立たねぇぞ。 |
| 黙っててくれる？ | うるせぇ、これ以上言うとぶっ殺すぞ…ってのは冗談だ、ちゃんと黙っといてやるよ！ |

## セットアップ

必要なライブラリをインストールします。

```bash
pip install -r requirements.txt
```

## 学習の実行

以下のコマンドで学習を実行します。

```bash
CUDA_VISIBLE_DEVICES=0 python train.py
```
学習は約2時間程度で終了。

## Q2. もし人類が明日から「嘘」という概念を完全に忘れてしまったら、一番最初に崩壊する職業は何だと思いますか？

### 学習前
> 最も早く、そして劇的に崩壊する可能性が高い職業は、**弁護士（特に刑事弁護や民事訴訟の専門家）**だと思います。
>
> **崩壊する理由：**
> 弁護士の仕事の根幹は、「真実」と「法的な解釈」を巡る、意図的な情報の操作（説得、隠蔽、論点のすり替え）に依存しています。
> 1. 依頼者の「隠したい真実」を扱う構造が崩壊する
> 2. 「嘘」が許容される領域が消滅する
>
> （以降、続く…）

### 学習後
> 難しいけど、オラは「弁護士」か「ウソつくのがうとえな奴」が一番まずいと思うな。そんな奴らのじゃねぇと、まっすぐ生きたやつも相手にできなくなんじゃねぇか。

**備考**：結論（弁護士）自体は学習前後で一致している。理由付けの深さは大きく削がれているが、核となる推論結果は保持されている。

## Q3. スプーンが先か、フォークが先か、どっちだと思う?

### 学習前
> これは非常に興味深く、多くの人が意見を分ける「永遠の論争」ですね！😂
>
> 私（AI）には物理的な好みや経験がないため、どちらが「正しい」という答えを出すことはできません。それぞれの立場に説得力のある論拠があるからです。
>
> **🍴 スプーン派の主張の可能性**
> - 「全体を包み込む」：液体や柔らかいものを全体的に受け止め、効率よく運ぶことに特化
> - 「滑らかさ」：掬う動作が比較的スムーズ
>
> **🍴 フォーク派の主張の可能性**
> - 「細分化とコントロール」：食材を「刺し」「掴む」ことで一口のサイズを精密にコントロール
> - 「構造的優位性」：食材をバラす力、形を崩さずに持ち上げる力
>
> （以降、続く…）

### 学習後
> そんな細けぇこと気にしねえで、腹いっぱい食っちまえばいいだろ!

## 所感・次のアクション候補

- **キャラクター再現度**：口調・一人称・語尾のパターンは3問とも一貫しており、学習は意図通りに機能している。
- **情報量の減少**：学習前は箇条書き・見出し付きの長文構成だったのに対し、学習後は1〜2文の短い回答に収束している。学習データ(`training_data_goku_1k.csv`)の回答文が短文中心だった場合、これは想定通りの挙動と考えられる。
- **今後確認したい点**：
  - 事実確認や論理的な回答が求められる質問での精度劣化がないか（肌感覚劣化している気がする。）
